# 留學生活平台 — 系統設計規格書 (Design Spec)

> **版本**: v1.1
> **建立日期**: 2026-03-25
> **專案代號**: StudyAbroad Hub

---

## 1. 產品概述

打造一站式留學準備與生活輔助平台，幫助留學生從申請選校、行前準備到落地生活的每個環節都能獲得可靠的資訊與社群支持。

### 目標用戶與核心價值
- **準留學生**（正在準備申請）：資訊透明，量化精準匹配。
- **在讀留學生**（已在海外就讀）：社群互助，資源共享。
- **畢業/校友留學生**（已畢業）：經驗分享，出售二手物品。

---

## 2. 核心功能模組

### 2.1 學校標準查詢 (School Explorer)
提供各學校的申請標準與歷屆錄取數據。
- 學校搜尋（名稱、排名、地區、科系）。
- 申請要求總覽（GPA、標化成績）。
- 歷屆錄取數據與截止日期追蹤。

### 2.2 背景落點分析 (Profile Match)
使用者輸入自身背景，系統自動分析推薦。
- 背景輸入表單（GPA、GRE/TOEFL、工作研究經歷）。
- 落點分析引擎（衝刺 / 適中 / 保底）。
- 契合度分析與個人化改善建議。
- AI 文書建議（需包含防抄襲檢測與免責聲明，並限制生成次數）。

### 2.3 留學地區資訊 (Area Guide)
提供學校所在地的生活資訊。
- 地區總覽、生活費估算、交通資訊、安全指數。
- **在地化處理**：單位自動切換（美制/公制），時區對齊顯示（學校當地時間 vs 用戶本地時間）。

### 2.4 留學準備工具 (Prep Kit) 與二手市集
行前準備清單與二手物品交易。
- 行李清單、租房指南。
- **二手市集**：強制建議面交與站內信溝通，不介入金流。需通過「認證用戶」權限方可發佈。

### 2.5 活動公告 (Events Board)
同區域留學生發佈與瀏覽活動。
- 活動發佈與分類標籤（需「認證用戶」權限）。
- RSVP 報名（「一般用戶」即可操作）。

## 3. 系統核心設計與營運策略

### 3.1 資料冷啟動與準確性 (Data Strategy)
- **種子資料 (Seed Data)**：Phase 1 內建「美國 Top 100 學校」與「英國 Top 50 學校」標準資料。
- **資料更新機制**：每年 8–9 月批次官方更新。開放社群「回報錯誤」，審核通過後發放「社群貢獻徽章」。

### 3.2 信任與安全機制 (Trust & Safety v2)
採用「分級與雙通道驗證」，兼顧防詐騙與準留學生的使用體驗：
- **一般用戶 (Basic User)**：
  - **獲取**：一般 Email/Google/Apple 註冊。
  - **權限**：無限使用查詢、落點分析、地區資訊。唯讀市集、可私訊賣家、可報名 (RSVP) 活動。
- **認證用戶 (Verified User)**：
  - **通道 A（自動）**：綁定有效 `.edu` 信箱驗證。
  - **通道 B（人工審核）**：上傳錄取通知書 (Offer Letter)、I-20 或舊學生證/成績單（針對已畢業或未取得信箱者）。
  - **專屬權限**：可發佈二手商品、可主辦社群活動。獲得「已認證」專屬標章。

### 3.3 用戶生命週期轉換 (Lifecycle Transition)
- **狀態自動轉換**：用戶於「落點分析」將某校標記為「已錄取 (Admitted)」並「決定就讀 (Enrolled)」後，首頁 Dashboard 自動切換為「行前準備模式」，主推該地區租房指南、新生接機與二手市集。

## 4. 技術架構與選型

- **Client Layer**: Next.js 15 (App Router) PWA 部署於 Vercel / Zeabur。後續加入 React Native。
- **API/BFF Layer**: 採用 Firebase 快速 MVP（Auth, Firestore, Cloud Functions），後續可視規模遷移至 GCP Cloud Run。
- **Data Layer**:
  - Cloud Firestore (主資料庫)。
  - Algolia (搜尋引擎)。

---

## 5. 資料模型 (核心 Entity)
*   **Schools**: 包含學校基本資訊、排名、各 Program 要求（GPA, 標化成績, 截止日）、學費。
*   **UserProfiles**: 記錄用戶背景數值（GPA, 標化成績, 目標學位）、認證狀態（Basic/Verified）。
*   **Listings** (二手商品): 綁定 `seller_uid` (需認證)、商品資訊、狀態。
*   **Events** (活動): 綁定 `organizer_uid` (需認證)、時間地點、RSVP 狀態。

---

## 6. 里程碑與時程 (Phase 1 MVP)
- **Month 1**: 需求確認與設計、前端架構搭建、Firebase 專案初始化。
- **Month 2**: 學校查詢模組開發、資料庫種子資料匯入。
- **Month 3**: 落點分析模組、分級認證機制實作。
- **Milestone**: Web MVP (Beta) 部署上線。
